1. WHAT WE'RE BUILDING
A dark-themed, fintech-style personal expense tracker — built with React, styled with Tailwind, powered by a live currency conversion API. The goal is not just to complete the brief but to build something that looks and feels like a real product.
Core Features
Add expenses: name, amount, category (Food / Travel / Marketing / Utilities / Other)
Expense list: card layout with category color, delete with exit animation
Running total: updates live as you add / remove entries
Category breakdown: visual progress bars per category
Currency conversion: live rates from Frankfurter API, 8 currency options
localStorage persistence: expenses survive page refresh
Responsive: works at 1600×900 (desktop) and 414×749 (mobile)

5. COMPONENT BREAKDOWN

📝  ExpenseForm
Controlled form for adding a new expense. Validates all fields before enabling submit. Resets and refocuses after successful add.
Props
  onAdd: (expense: { name, amount, category }) => void
State / Logic
• Local state: { name, amount, category } — all strings initially empty
• isValid = name.trim() && parseFloat(amount) > 0 && category !== ""
• useRef on name input → focus() after submit
• onSubmit: call onAdd(), reset state, refocus
UX Details
✓  Category select shows emoji inline (🍔 Food, ✈️ Travel, etc.)
✓  Submit button disabled + opacity-50 when isValid is false
✓  Amount input: type="number" min="0.01" step="0.01"
✓  New card entering the list should animate: fade-in + slide-down

🃏  ExpenseCard
Renders a single expense. Handles its own deletion animation — waits 280ms before calling onDelete so the exit animation completes.
Props
  expense: { id, name, amount, category, createdAt }
  onDelete: (id: string) => void
State / Logic
• Local: deleting = false → set true on delete click
• setTimeout 280ms → then call onDelete(expense.id)
• className includes "opacity-0 scale-95" when deleting is true
UX Details
✓  Category color dot on the left (bg-[category.color] w-2 h-2 rounded-full)
✓  Amount always in font-mono — right-aligned
✓  Category badge: small pill with category.color background at 20% opacity
✓  Hover state on card: subtle border brightening
✓  Delete button: hidden by default, visible on card hover (group-hover:visible)

📋  ExpenseList
Renders sorted list of ExpenseCards. Handles empty state and exposes a sort toggle. Passes onDelete down to each card.
Props
  expenses: Expense[]
  onDelete: (id: string) => void
State / Logic
• Local: sortBy = "date" | "amount"
• sorted = useMemo: if "amount" → [...expenses].sort((a,b) => b.amount - a.amount)
• Otherwise expenses array is already newest-first (from useExpenses hook)
UX Details
✓  Empty state: centered icon + "No expenses yet. Add your first one above."
✓  Sort toggle: two small ghost buttons — "Latest" | "Highest"
✓  Expense count badge in section header: "3 expenses"
✓  List scrollable independently on desktop (max-h + overflow-y-auto)

📊  SummaryPanel
Shows the running total, converted total, and animated category breakdown bars. Receives all data as props — no internal fetching.
Props
  total: number
  byCategory: Record<string, number>
  convertedTotal: number | null
  targetCurrency: string
State / Logic
• maxSpend = Math.max(...Object.values(byCategory)) for bar scaling
• barWidth = (categorySpend / maxSpend) * 100 + "%"
• No local state — pure display component
UX Details
✓  Total: font-mono text-4xl lg:text-5xl — most prominent element
✓  Converted total shown below in muted text if convertedTotal !== null
✓  Category bars animate width: transition-[width] duration-500 ease-out
✓  Each bar row: emoji + label left, amount right, bar underneath
✓  "Highest spend" accent badge on top category
✓  If no expenses yet, show placeholder dashes instead of zeros

💱  CurrencyConverter
Fetches live exchange rates from Frankfurter API. Exposes a currency dropdown. Notifies parent of the converted total via onConverted callback.
Props
  total: number
  onConverted: (amount: number | null, currency: string) => void
State / Logic
• Local: targetCurrency = "INR" (default — good for Indian audience)
• useCurrencyRate(targetCurrency) custom hook handles fetch
• converted = rates?.[targetCurrency] ? (total * rates[targetCurrency]).toFixed(2) : null
• useEffect: call onConverted(converted, targetCurrency) when either changes
UX Details
✓  Loading: skeleton pulse div where the amount would be
✓  Error: amber banner "Live rates unavailable" + Retry button — component stays visible
✓  "Updated at HH:MM" timestamp shown in muted text on success
✓  Currencies: USD, EUR, GBP, INR, JPY, AED, SGD, CAD
✓  Dropdown styled to match the dark theme (not browser default)

6. CUSTOM HOOKS
useExpenses.js
Encapsulates 100% of expense business logic. App.jsx only calls this hook and passes results to components.
export function useExpenses() {
  // ① Lazy initializer reads localStorage once on mount
  const [expenses, setExpenses] = useState(() => {
    try {
      const saved = localStorage.getItem("mm-expenses");
      return saved ? JSON.parse(saved) : [];
    } catch { return []; }
  });

  // ② Persist to localStorage on every change
  useEffect(() => {
    localStorage.setItem("mm-expenses", JSON.stringify(expenses));
  }, [expenses]);

  // ③ CRUD
  const addExpense = ({ name, amount, category }) =>
    setExpenses(prev => [{ id: uuid(), name, amount: parseFloat(amount),
      category, createdAt: Date.now() }, ...prev]);

  const deleteExpense = (id) =>
    setExpenses(prev => prev.filter(e => e.id !== id));

  // ④ Derived state — memoized, never re-computed unless expenses changes
  const total = useMemo(
    () => expenses.reduce((sum, e) => sum + e.amount, 0), [expenses]);

  const byCategory = useMemo(() =>
    expenses.reduce((acc, e) => {
      acc[e.category] = (acc[e.category] || 0) + e.amount;
      return acc;
    }, {}), [expenses]);

  return { expenses, addExpense, deleteExpense, total, byCategory };
}

useCurrencyRate.js
Handles all API concerns. Always returns loading/error/data — never leaves the caller guessing.
export function useCurrencyRate(baseCurrency = "USD") {
  const [rates, setRates]         = useState(null);
  const [loading, setLoading]     = useState(false);
  const [error, setError]         = useState(null);
  const [updatedAt, setUpdatedAt] = useState(null);

  const fetchRates = useCallback(async () => {
    setLoading(true); setError(null);
    try {
      const res = await fetch(
        `https://api.frankfurter.app/latest?from=${baseCurrency}`
      );
      if (!res.ok) throw new Error("API responded with " + res.status);
      const data = await res.json();
      setRates(data.rates);
      setUpdatedAt(new Date());
    } catch (err) {
      setError("Could not fetch live rates.");
      // Important: do NOT clear previous rates — show stale data if available
    } finally {
      setLoading(false);
    }
  }, [baseCurrency]);

  useEffect(() => { fetchRates(); }, [fetchRates]);

  return { rates, loading, error, updatedAt, retry: fetchRates };
}

7. APP.JSX — THE COMPOSITION LAYER
App.jsx should have ZERO business logic. It only wires components together. This is the single biggest signal of developer maturity that reviewers look for.
export default function App() {
  // One hook call — all expense logic lives inside it
  const { expenses, addExpense, deleteExpense, total, byCategory }
    = useExpenses();

  // Only UI state lives here
  const [convertedTotal, setConvertedTotal] = useState(null);
  const [targetCurrency, setTargetCurrency] = useState("INR");

  return (
    <div className="min-h-screen bg-[#0A0A0A] text-white font-sans">

      {/* Header */}
      <header className="border-b border-[#1A1A1A] px-6 py-4 flex justify-between items-center">
        <h1 className="font-display text-xl font-bold text-accent">
          💸 ExpenseTrack
        </h1>
        <span className="font-mono text-2xl font-semibold">
          ${total.toFixed(2)}
        </span>
      </header>

      {/* Two-column grid */}
      <main className="max-w-7xl mx-auto px-4 py-8
                       grid grid-cols-1 lg:grid-cols-[400px_1fr] gap-6">

        <aside className="flex flex-col gap-6">
          <ExpenseForm onAdd={addExpense} />
          <CurrencyConverter
            total={total}
            onConverted={(amt, cur) => {
              setConvertedTotal(amt);
              setTargetCurrency(cur);
            }}
          />
        </aside>

        <section className="flex flex-col gap-6">
          <SummaryPanel
            total={total}
            byCategory={byCategory}
            convertedTotal={convertedTotal}
            targetCurrency={targetCurrency}
          />
          <ExpenseList expenses={expenses} onDelete={deleteExpense} />
        </section>

      </main>
    </div>
  );
}
